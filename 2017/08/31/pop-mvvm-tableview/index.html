Protocol-Oriented Programming 与 MVVM 实践中的TableView

又来混一篇blog，最近可能真的对面向协议编程上瘾了，写啥都想先看看能不能用协议来实现。在上一篇Protocol-Oriented Programming 与 MVVM 实践 | BBBBB’s Land的基础上，这两天又进行了一次还算成功的踩坑。

这次涉及到的对象是TableView。

之前的文章中提到过，我之前项目的ViewController和ViewModel都是继承的，自然而然，我会有一个TableViewController作为项目中所有TableViewController的基类，来处理一些公共的代码：

正如以上代码所示，秉承着少写重复代码的原则，我把所有TableView都会执行的一些公共方法全都放在了基类执行，因为这些方法的行为在子类中也基本不会有大的改变。

但在新项目中，我一开始就打算尽量不使用继承，但是当我开始写TableViewControler之后，才发现，像UITableViewDataSource里面的一些方法，要是不是用继承，那么就得在每个类里面都在实现一遍，我这么懒的人，这不能忍。所以我打算，在适当的程度下，还是稍微使用一下继承。毕竟继承也是著名的面向对象编程三大特性之一，也不能完全的批判一番。

思路很快就有了：同样写一个TableViewController的基类，同样把UITableViewDelegate和UITableViewDataSource的几个方法放在基类中实现。基类除了遵循UITableViewDelegate和UITableViewDataSource之外，不遵循任何其他的协议。在具体的子类中，再遵循我们之前定义的RoutableViewControllerType等MVVM中所需要的协议。
