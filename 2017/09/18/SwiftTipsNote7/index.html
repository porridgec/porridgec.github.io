随机数生成

随机数是老生常谈的问题了，在 Swift 中它又焕发出了新的坑，即我们在 Objective-C 中常用的 arc4random() ，不能像以前一样使用了。

因为 Swift 中的 Int ，是跟 CPU 的架构相关的，在32位的 CPU 上面它是 Int32 ，而在64位的 CPU 上面它是 Int64 。而 arc4random() 返回值永远是 UInt32 ，所以在32位的机器上，有一半的几率在 Int 的类型转换时越界。

Log 输出

在 Swift 中，打log最普通的方法就是使用 print() 函数了，当然，我们可能需要添加一下额外的信息，比如当前文件，当前行数等，这时候需要用到一下符号：

符号 类型 描述
#file String 文件路径
#line Int 当前行数
#column Int 当前列数
#function String 当前函数名

宏定义 define

首先，需要明确的是 #define 在 Swift 已经彻底被去除了。

以下是官方给出的替换建议：

  • 使用合适作用范围的 let 或 get 属性代替通过 #define 定义的值
  • 对于 #define 定义的方法，在相同的作用域重新写一个对应的方法
